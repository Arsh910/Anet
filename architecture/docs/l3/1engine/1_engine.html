engine.py — the conductor.

anet/core/engine.py is the file that owns one turn of the conversation. Every You: ... in your terminal hits Engine.run_turn() here and doesn't return until the synthesizer has streamed the answer back. 1141 lines, broken into 5 sub-pages so each section gets room to breathe.

Where it sits

Engine is the top of the dependency tree — almost every other core module is something it calls. The arrows below point in the direction of "uses": engine uses orchestrator, orchestrator uses agent_runner, and so on.

Reading order recap: we explain engine.py first because it's where execution starts. Each time engine references another file (orchestrator, memory_store, context_window, …) we'll cover that file in its own chapter — see the L3 index.

Why the lazy imports? Two reasons. (1) Several of these modules — especially memory_store and config_loader — touch the filesystem on import (loading YAML, opening Chroma). Lazy-importing them inside the functions that need them keeps engine importable even when the workspace isn't fully set up yet. (2) Circular-import avoidance: orchestrator imports things that ultimately reach back to engine-adjacent helpers.
