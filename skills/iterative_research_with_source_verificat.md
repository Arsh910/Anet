## Iterative Research with Source Verification
**Applies to:** Research tasks requiring information gathering from multiple sources with verification, especially when initial searches are too broad or lack authoritative answers
**Steps:**
1. Start with a broad search on the main topic to understand the information landscape
2. Analyze initial results to identify gaps, ambiguities, or lack of authoritative sources
3. Refine search with more specific keywords, technical terms, or pattern names discovered in step 1
4. Prioritize authoritative sources (official documentation, established tech blogs, recent articles from trusted platforms)
5. Cross-reference information across 2-3 independent sources to verify accuracy
6. Extract specific answers, patterns, or data points requested in the task
7. If conflicting information exists, note the discrepancies and weight sources by recency and authority
8. Compile findings in a structured format that directly addresses the original task requirements
**Notes:** Always check publication dates - prioritize sources from the last 1-2 years for fast-moving tech topics. When verifying technical patterns, prefer official documentation (e.g., Python docs) over third-party tutorials. If searches return incomplete information, try alternative phrasings or related technical terms.
