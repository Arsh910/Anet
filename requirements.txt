# ── Long-term memory (memory_tool / memory_store) ───────────────────────────────
# mem0 is the memory backend: a local Chroma vector store + on-device fastembed
# embeddings (ONNX, no torch, no API key — one-time ~130 MB model download on first
# use) + the user's configured LLM for fact extraction/de-duplication. No server and
# no hosted service; embeddings are computed locally and the vector DB is a folder
# under ~/.anet/memory/. If mem0 can't be imported, long-term memory simply disables
# itself and the rest of ANet keeps working.
mem0ai
chromadb
fastembed

# ── RecMem memory backend (memory.backend: recmem — the DEFAULT) ────────────────
# Native 3-tier recurrence memory under anet/memory/recmem/ (subconscious→episodic→
# semantic). The LLM is invoked only to consolidate when an interaction RECURS — far
# cheaper than mem0's per-interaction extraction. Reuses the chromadb + fastembed
# already listed above for embeddings + the vector store (no qdrant, no extra
# embedding service). No hard extra deps.
#   json-repair — OPTIONAL: tolerant parsing of the LLM's JSON consolidation output.
#                 Without it, RecMem falls back to stdlib json + a regex extractor.
json-repair
